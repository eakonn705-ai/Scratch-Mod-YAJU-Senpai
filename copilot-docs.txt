■ Scratchの基本
Scratchは、MITメディアラボが開発したビジュアルプログラミング環境（2007年公開）。
対象は主に8〜16歳だが年齢制限はなく、公式サイトはscratch.mit.edu。
プロジェクトはブロックを組み合わせてスプライト・ステージを制御する。
ブラウザ上で動作し、sb3形式（ZIPベース）でプロジェクトを保存・共有できる。

■ ステージとスプライト
【ステージ】
480×360ピクセルの実行エリア。座標系はX軸-240〜240、Y軸-180〜180で中心が(0,0)。
背景（バックドロップ）を複数持てる。ステージ自体もスクリプトを持てるが動きブロックは使えない。

【スプライト】
ステージ上のオブジェクト。複数のコスチューム（見た目）と音を持てる。
各スプライトは独立したスクリプトを持ち、同時並行で動く。
プロパティ: X座標、Y座標、向き（90=右、-90=左、0=上、180=下）、サイズ(%)、表示/非表示、レイヤー順。
スプライトのクローンを作成でき、クローンは独立して動作する。

■ ブロックカテゴリ一覧

【動き】スプライトの位置・向きを操作する
- 「○歩動かす」: 向いている方向に指定歩数移動。
- 「○度回す（時計回り/反時計回り）」: 向きを回転。
- 「○度に向ける」: 絶対角度で向きを設定。
- 「マウスのポインター/スプライトへ向ける」: 対象の方向を向く。
- 「x座標を○にする / y座標を○にする」: 絶対座標で移動。
- 「x座標を○ずつ変える / y座標を○ずつ変える」: 相対移動。
- 「○秒でx:○y:○に移動する」: 指定秒数かけて滑らかに移動。
- 「もし端に着いたら、跳ね返る」: 画面端で反射。
- 「回転方法を○にする」: 左右のみ/自由に回転/回転しない の3択。

【見た目】コスチューム・表示・エフェクトを操作する
- 「○と言う / ○秒間○と言う」: 吹き出しを表示。
- 「次のコスチュームにする / コスチュームを○にする」: 見た目を切り替え。
- 「次の背景にする / 背景を○にする」: 背景を切り替え。
- 「サイズを○%にする / ○%ずつ変える」: 拡大縮小。
- 「色/魚眼/渦巻き/ピクセル化/モザイク/明るさ/幽霊の効果を○にする」: 各種エフェクト。
- 「表示する / 隠す」: スプライトの表示切替。
- 「最前面に出る / 最背面に出る / ○番前に出る」: レイヤー操作。

【音】音の再生・音量・テンポを操作する
- 「○の音を鳴らす」: 音を再生（次のブロックへ続く）。
- 「最後まで○の音を鳴らす」: 音の終了まで待機。
- 「音量を○%にする / ○%ずつ変える」: 音量調整。

【イベント】スクリプトの起動条件を定義する
- 「緑の旗が押されたとき」: プロジェクト起動時のトリガー。
- 「○キーが押されたとき」: キーボード入力のトリガー。
- 「このスプライトが押されたとき」: クリックのトリガー。
- 「背景が○になったとき」: 背景変化のトリガー。
- 「○を受け取ったとき」: メッセージ受信のトリガー。
- 「○を送る」: メッセージを送信してスクリプトを続ける。
- 「○を送って待つ」: 全受信側の処理完了まで待機してから続く。

【制御】フロー制御を行う
- 「○秒待つ」: 処理を一時停止。
- 「○回繰り返す」: 指定回数ループ。
- 「ずっと」: 無限ループ。
- 「もし○なら」: 条件分岐（if）。
- 「もし○なら / でなければ」: if-else。
- 「○まで待つ」: 条件が真になるまで待機。
- 「○まで繰り返す」: 条件が真になるまでループ。
- 「このスクリプトを止める / すべてを止める / スプライトの他のスクリプトを止める」: 実行停止。
- 「クローンを作る（自分自身/○）」: クローン生成。
- 「クローンされたとき」: クローン起動のトリガー。
- 「このクローンを削除する」: クローン消去。

【調べる】入力・センサー・環境情報を取得する
- 「○に触れた?」: スプライト/端/マウスへの接触判定（ブール値）。
- 「色○に触れた? / 色○が色○に触れた?」: 色接触判定。
- 「○までの距離」: 対象までの距離を取得。
- 「○と聞いて待つ」: テキスト入力を受け取り「答え」変数に格納。
- 「○キーが押された?」: キー入力判定（ブール値）。
- 「マウスが押された?」: マウスボタン判定。
- 「マウスのx座標 / y座標」: マウス位置取得。
- 「タイマー」: 経過時間（秒）を取得。
- 「タイマーをリセット」: タイマーを0にリセット。
- 「現在の年/月/日/曜日/時/分/秒」: 現在時刻を取得。

【演算子】計算・比較・論理・文字列操作を行う
- 算術: 「○+○」「○-○」「○*○」「○/○」
- 比較: 「○<○」「○=○」「○>○」（ブール値を返す）
- 論理: 「○かつ○」「○または○」「○でない」
- 「○から○までの乱数」: 範囲内のランダム整数（両端含む）。
- 「○の絶対値/切り捨て/切り上げ/四捨五入/平方根/sin/cos/tan」: 数学関数。
- 「○と○」: 文字列連結。
- 「○の長さ」: 文字列長。
- 「○の○番目の文字」: 文字列インデックス（1始まり）。
- 「○に○が含まれる?」: 文字列検索（ブール値）。

【変数】値を記憶・管理する
- 変数は「すべてのスプライト用」（グローバル）か「このスプライトのみ」（ローカル）で作成。
- 「○を○にする」: 変数に値をセット。
- 「○を○ずつ変える」: 変数に加算。
- 「○を表示する / 隠す」: モニター表示切替。

【リスト】配列的なデータ構造を扱う
- 要素は1始まりのインデックスでアクセスする。
- 「○に○を追加する」: 末尾に要素追加。
- 「○の○番目を削除する / すべて削除する」: 要素削除。
- 「○の○番目に○を挿入する」: 指定位置に挿入。
- 「○の○番目を○にする」: 要素の上書き。
- 「○の○番目」: 要素取得。
- 「○の長さ」: リストの要素数。
- 「○に○が含まれる?」: 要素検索（ブール値）。

【自分で作る（カスタムブロック）】
- 独自のブロックを定義できる。引数（数値・文字列・ブール）を持てる。
- 「画面を再描画せずに実行する」オプションで高速ループが可能。

■ 拡張機能
【ペン】描画機能。「ペンを下ろす/上げる」「ペンの色/太さを○にする」「スタンプ」「消す」。
【音楽】楽器演奏。「楽器を○にする」「○の音符を○拍鳴らす」「○拍休む」。
【テキスト読み上げ（TTS）】「○を読み上げる」「声を○にする」「言語を○にする」。
【翻訳】「○を○語に翻訳する」。Night-ModではGAS（Google Apps Script）のJSONP経由で動作。
【ビデオモーション検出】カメラを使ったモーション検出。

■ よく使うアルゴリズム・テクニック

【ゲームループ】
「緑の旗が押されたとき → ずっと → （ゲームロジック）」
FPSを固定したい場合は末尾に「0.016秒待つ」を入れる。

【滑らかな動き（慣性）】
変数「速度X」「速度Y」を用意し、毎フレーム「X座標をX座標+速度Xにする」。
速度に摩擦係数（例:0.9）を掛けることで徐々に減速する。

【8方向キャラクター操作】
「ずっと」内で各矢印キーの判定をif文で記述。
例:「もし右矢印キーが押された?なら → x座標を5ずつ変える」

【当たり判定】
シンプルな判定:「もし○に触れた?なら」
精度が必要な場合は色判定:「もし色○に触れた?なら」（重いので注意）

【クローンによる弾・敵の生成】
親スプライト:「ずっと → クローンを作る → 0.5秒待つ」
クローン:「クローンされたとき → 動き処理 → もし端に触れたら → このクローンを削除する」

【タイマー処理】
「緑の旗が押されたとき → タイマーをリセット → ずっと → もしタイマー>制限時間なら → ゲームオーバー処理」

【メッセージによるシーン管理】
「ゲーム開始を送る」「ゲームオーバーを送る」でシーンを切り替え。
受信側のスクリプトで初期化処理を行う。

【ランダム配置】
「x座標を-240から240までの乱数にする」
「y座標を-180から180までの乱数にする」

【sin/cosを使った円運動】
変数「角度」を毎フレーム加算し、
「x座標を 中心X + 半径*([演算子]cosの角度) にする」
「y座標を 中心Y + 半径*([演算子]sinの角度) にする」

【リストを使ったスタック・キュー】
スタック（後入れ先出し）: 末尾に追加・末尾から取得して削除。
キュー（先入れ先出し）: 末尾に追加・先頭(1番目)から取得して削除。

■ Night-Mod固有の機能
- 通常のScratchと同じsb3ファイルをブラウザ上で実行できるカスタム環境。
- FPSを30〜120の範囲で変更可能（オーバークロック機能）。
- 翻訳拡張機能はGAS（Google Apps Script）のJSONP経由で動作する。
- リストモニターはスクロール可能（通常のScratchでは不可）。
- 背景画像をnight-mod.png.jpgというファイル名で差し替え可能。
- sb3ファイルを読み込むと、Scratch Copilotがそのプロジェクトの内容を認識する。

■ よくあるバグと対処法
- 音が出ない: ブラウザの自動再生ポリシーにより、最初のクリック後から有効になる。
- 画面が真っ黒: 緑の旗を押すかsb3を再読み込みする。
- クローンが増えすぎる: Scratchのクローン上限は300個。「このクローンを削除する」を忘れずに。
- 変数が動かない: グローバル変数かローカル変数かを確認する。
- 当たり判定がずれる: スプライトの中心点（コスチュームエディタで設定）を確認する。
