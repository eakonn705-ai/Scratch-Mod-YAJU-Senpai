USER GUIDE

01. プロジェクトの読込

「Choose File」ボタンをクリックし、パソコン上の .sb3 ファイルを選択してください。ファイルを選ぶと自動的に読み込みが始まり、完了するとステータスが CORE ACTIVE に変わります。
読み込みに成功すると、プロジェクトは自動的に緑の旗が押された状態で起動します。

02. 実行と停止

▶ 緑の旗ボタン：プロジェクトを最初から実行します。すでに動いている場合はリスタートされます。
⬡ 赤い停止ボタン：実行中のすべてのスクリプトを強制停止します。音も止まります。

03. FPS オーバークロック

スライダーで実行フレームレートを 30〜120 FPS の範囲で調整できます。デフォルトは通常の Scratch と同じ 30 FPS です。
数値を上げると演算・アニメーション・物理が高速になります。ただし、wait ブロックや タイマー は実時間ベースのため影響を受けません。高FPS化を前提としていないプロジェクトでは動作が乱れる場合があります。

04. キーボード操作

キーボード入力はステージキャンバスにフォーカスが当たっている状態で動作します。
対応キー：Space、矢印キー（↑↓←→）、英数字キー など Scratch 標準のすべてのキー。
テキスト入力（ask ブロック）中はゲームへのキー送信が自動的に抑制されるため、回答入力と操作が干渉しません。

05. マウス操作

ステージ上でのマウス移動・クリックは Scratch の座標系（X: −240〜240、Y: −180〜180）に変換されてVMへ送信されます。スプライトのクリック判定やマウス追従なども通常通り動作します。

06. 翻訳拡張ブロック

Scratch の「翻訳」拡張ブロックが使えます。通常の Scratch サーバーは外部からブロックされますが、Night-Mod では Google Apps Script（GAS） 経由の JSONP 方式でリクエストを迂回し、翻訳を実現しています。
翻訳が返ってこない場合、GAS のデプロイ設定で「アクセスできるユーザー」を 「全員（匿名ユーザーを含む）」 に設定してください。

07. 変数・リストモニター

プロジェクト内で表示設定にした変数やリストは、ステージ上に重なって表示されます。
リストはアイテム数が多い場合でもスクロール可能です。リスト内でマウスホイールを回すかドラッグしてスクロールしてください。変数は通常・大きい・スライダーの各表示モードに対応しています。

08. ask ブロック（質問入力）

「〇〇と聞いて待つ」ブロックが実行されると、ステージ下部に入力フォームが表示されます。テキストを入力して Enter キー または送信ボタンをクリックすると、答えがVMへ渡されてスクリプトが続行されます。

09. 背景画像のカスタマイズ

Night-Mod のインターフェース背景は night-mod.png.jpg というファイル名の画像を参照しています。このファイルを差し替えることで、好きな背景画像に変更できます。

10. ステータスバーの見方

CONNECTING… : ライブラリ読み込み中
SYNCING… : VM初期化リトライ中
CORE ACTIVE : 正常起動・待機中
UPLOADING… : sb3 ファイル読み込み中
ERROR / FAILURE : 初期化・読込エラー

11. 対応拡張機能

プロジェクト内に含まれる拡張機能は、読み込み時に自動検出してロードを試みます。
正式対応：翻訳、テキスト読み上げ（TTS）、ペン、音楽、ビデオモーション検出 など。ただし一部の非公式拡張はロードできない場合があります。

12. トラブルシューティング

音が出ない：ブラウザの自動再生ポリシーにより、最初の操作（クリックやキー押下）の後から音が有効になります。
画面が真っ黒：緑の旗を押してプロジェクトを起動してください。sb3 を再度読み込む方法も有効です。
翻訳が動かない：GAS のデプロイ設定を確認してください（項目06を参照）。
読込エラー：有効な .sb3 ファイルか確認してください。破損したファイルや未対応形式は読み込めません。

Scratch Copilot について
